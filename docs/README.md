## Установка и запуск

1. Скопировать файл окружения и задать нужные значения:

   ```bash
   cp .env.example .env
   ```

2. Запустить контейнеры:

   ```bash
   docker-compose up -d
   ```

API будет доступен по адресу `http://localhost:8080`.

## Модели данных

### Department (Отдел)

```json
{
  "id": 1,
  "name": "Разработка",
  "parent_id": null,
  "depth": 1,
  "created_at": "2024-01-01T00:00:00Z",
  "children": [],
  "employees": []
}
```

| Поле         | Тип        | Описание                                        |
|--------------|------------|-------------------------------------------------|
| `id`         | number     | Идентификатор                                   |
| `name`       | string     | Название отдела                                 |
| `parent_id`  | number\|null | ID родительского отдела (null — корневой)     |
| `depth`      | number     | Уровень вложенности (1 = корневой, макс. 5)    |
| `created_at` | string     | Дата создания (ISO 8601)                        |
| `children`   | array      | Дочерние отделы (только при запросе с `depth`) |
| `employees`  | array      | Сотрудники (только при `include_employees=true`) |

### Employee (Сотрудник)

```json
{
  "id": 1,
  "department_id": 2,
  "full_name": "Иванов Иван Иванович",
  "position": "Разработчик",
  "hired_at": "2023-06-01",
  "created_at": "2023-06-01T10:00:00Z"
}
```

| Поле            | Тип        | Описание                        |
|-----------------|------------|---------------------------------|
| `id`            | number     | Идентификатор                   |
| `department_id` | number     | ID отдела                       |
| `full_name`     | string     | Полное имя                      |
| `position`      | string     | Должность                       |
| `hired_at`      | string\|null | Дата приёма на работу (YYYY-MM-DD) |
| `created_at`    | string     | Дата создания записи (ISO 8601) |

## API: Отделы

### GET /departments

Возвращает список корневых отделов (глубина 1).

**Ответ `200 OK`:**
```json
[
  { "id": 1, "name": "Разработка", "parent_id": null, "depth": 1, "created_at": "..." },
  { "id": 2, "name": "Маркетинг",  "parent_id": null, "depth": 1, "created_at": "..." }
]
```

---

### GET /departments/{id}

Возвращает отдел по ID.

**Query-параметры:**

| Параметр            | Тип     | Описание                                                              |
|---------------------|---------|-----------------------------------------------------------------------|
| `depth`             | integer | Глубина загрузки дочерних отделов (0 = без детей, по умолчанию `0`) |
| `include_employees` | boolean | Включить сотрудников отдела (`true` / `false`)                       |

**Пример:**
```
GET /departments/1?depth=2&include_employees=true
```

**Ответ `200 OK`:**
```json
{
  "id": 1,
  "name": "Разработка",
  "parent_id": null,
  "depth": 1,
  "created_at": "...",
  "children": [
    {
      "id": 3,
      "name": "Backend",
      "parent_id": 1,
      "depth": 2,
      "created_at": "...",
      "children": []
    }
  ],
  "employees": [
    { "id": 1, "full_name": "Иванов И.И.", "position": "Тимлид", ... }
  ]
}
```

---

### POST /departments

Создаёт новый отдел.

**Тело запроса:**
```json
{
  "name": "Backend",
  "parent_id": 1
}
```

| Поле        | Тип         | Обязательное | Описание                                     |
|-------------|-------------|:------------:|----------------------------------------------|
| `name`      | string      | Да           | Название отдела                              |
| `parent_id` | number\|null | Нет         | ID родительского отдела (null — корневой)   |

**Ответ `201 Created`:** объект созданного отдела.

---

### PATCH /departments/{id}

Обновляет название и/или положение отдела в иерархии.

**Тело запроса** (минимум одно поле обязательно):
```json
{
  "name": "Новое название",
  "parent_id": 5,
  "move_to_root": false
}
```

| Поле           | Тип         | Описание                                                           |
|----------------|-------------|--------------------------------------------------------------------|
| `name`         | string      | Новое название                                                     |
| `parent_id`    | number      | Переместить в другой родительский отдел                           |
| `move_to_root` | boolean     | `true` — сделать отдел корневым (снять привязку к родителю)       |

**Ответ `200 OK`:** обновлённый объект отдела.

---

### DELETE /departments/{id}

Удаляет отдел.

По умолчанию удаление завершается ошибкой, если в отделе есть сотрудники — их нужно предварительно перевести в другой отдел или использовать режим `reassign`.

**Query-параметры:**

| Параметр                       | Тип     | Описание                                             |
|-------------------------------|---------|------------------------------------------------------|
| `mode`                        | string  | `reassign` — перевести сотрудников перед удалением  |
| `reassign_to_department_id`   | number  | ID отдела для перевода (обязательно при `mode=reassign`) |

**Пример:**
```
DELETE /departments/3?mode=reassign&reassign_to_department_id=7
```

**Ответ `204 No Content`.**

---

### POST /departments/{id}/employees

Создаёт сотрудника в указанном отделе.

**Тело запроса:**
```json
{
  "full_name": "Иванов Иван Иванович",
  "position": "Разработчик",
  "hired_at": "2023-06-01"
}
```

| Поле        | Тип    | Обязательное | Описание                            |
|-------------|--------|:------------:|-------------------------------------|
| `full_name` | string | Да           | Полное имя                          |
| `position`  | string | Да           | Должность                           |
| `hired_at`  | string | Нет          | Дата приёма (формат `YYYY-MM-DD`)  |

**Ответ `201 Created`:** объект созданного сотрудника.

## API: Сотрудники

> Эндпоинты для управления сотрудниками напрямую (без привязки к отделу) в разработке.

---

## Бизнес-правила

1. **Максимальная глубина иерархии — 5.** Создание или перемещение отдела на 6-й уровень и глубже запрещено.
2. **Уникальность названия среди соседних отделов.** Два отдела с одним и тем же родителем не могут иметь одинаковое название.
3. **Каскадное удаление.** При удалении отдела все его дочерние отделы и сотрудники удаляются вместе с ним (если не использован режим `reassign`).
4. **Автоматическое обновление глубины.** При перемещении отдела значение `depth` пересчитывается у него и всех его потомков.
5. **Отдел не может быть собственным родителем.**
6. **Отдел нельзя переместить внутрь собственного поддерева** (защита от циклических ссылок).
7. **Перевод сотрудников при удалении.** Режим `mode=reassign` атомарно переводит всех сотрудников удаляемого отдела в указанный и затем удаляет отдел.

---

## Коды ошибок

| HTTP-код | Причина                                                     |
|----------|-------------------------------------------------------------|
| `400`    | Некорректные параметры запроса или тела                     |
| `404`    | Отдел или сотрудник не найден                               |
| `409`    | Название отдела уже занято среди соседних                   |
| `422`    | Нарушение бизнес-правила (например, превышена глубина)      |
| `500`    | Внутренняя ошибка сервера                                   |

**Формат ошибки:**
```json
{
  "error": "department name already exists in this parent"
}
```
